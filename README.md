# Private OCR to Markdown

完全ローカル処理のOCR to Markdown変換アプリケーション

## 🌐 デモ

**[https://private-ocr-markdown.vercel.app/](https://private-ocr-markdown.vercel.app/)**

ブラウザで今すぐ使えます！インストール不要で、すべての処理がブラウザ内で完結します。

## 特徴

- **完全プライベート**: ファイルやOCR結果を外部サーバーに送信しません
- **高速**: ブラウザ内で完結（インストール不要）
- **使えるMarkdown**: 見出し、箇条書き、表、コードブロックを自動整形
- **テンプレート対応**: 議事録、論文、請求書などの用途別出力形式

## 技術スタック

- **Framework**: Next.js 14+ (App Router)
- **OCR Engine**: Tesseract.js
- **PDF Library**: pdf.js
- **State Management**: Zustand
- **UI/Design**: Tailwind CSS + Lucide React
- **Language**: TypeScript

## 📖 使い方

### 基本的な使い方

1. **[デモサイト](https://private-ocr-markdown.vercel.app/)にアクセス**
   - ブラウザで開くだけでOK（インストール不要）

2. **ファイルをアップロード**
   - 画像（PNG, JPG, WebP）またはPDFをドラッグ&ドロップ
   - または「クリックしてファイルを選択」から選択
   - 複数ファイルの同時アップロード可能

3. **設定を選択**
   - **認識言語**:
     - `日本語 + 英語` - 日英混在の文書に最適（推奨）
     - `日本語のみ` - 日本語のみの文書
     - `英語のみ` - 英語のみの文書
   - **出力形式**:
     - `議事録` - 会議の記録、日時・議題・決定事項を自動強調
     - `論文・レポート` - Abstract、Introductionなどの章立てを強調
     - `請求書・帳票` - 発行元、宛先、明細などを構造化

4. **OCR処理を開始**
   - 「OCR処理を開始」ボタンをクリック
   - 進捗がリアルタイムで表示されます
   - 処理には数秒〜数十秒かかります（ファイルサイズによる）
   - 途中でキャンセルも可能

5. **結果を確認・エクスポート**
   - 処理が完了すると、Markdownプレビューが表示されます
   - **「Markdownをコピー」** - クリップボードにコピーして、他のアプリに貼り付け
   - **「ダウンロード (.md)」** - Markdownファイルとして保存

### 💡 使用例

- **会議の議事録作成**: スマホで撮影したホワイトボードの写真からMarkdown議事録を生成
- **論文のデジタル化**: スキャンしたPDF論文をMarkdown形式に変換してNotionに保存
- **請求書の整理**: 紙の請求書をスキャンして、Markdownテーブルで管理
- **名刺のデジタル化**: 名刺の写真から連絡先情報を抽出

### ⚠️ 注意事項

- **初回起動時**: OCRエンジンの学習データ（約10-20MB）をダウンロードするため、少し時間がかかります
- **大きなファイル**: ファイルサイズが大きい場合、処理に時間がかかることがあります
- **手書き文字**: 手書き文字の認識精度は印刷文字より低くなります
- **画質**: 画質が悪いとOCR精度が低下します。可能な限り高解像度の画像を使用してください

## 主な機能

### OCR処理

- Tesseract.jsを使用したブラウザ内OCR
- Web Workerによる非ブロッキング処理
- ページごとの進捗表示
- キャンセル機能

### Markdown変換

- **座標ベース変換**: OCR結果の座標情報（Bounding Box）を活用
- **見出し検出**: 行高と前後の余白から見出しレベルを判定
- **箇条書き検出**: 行頭記号とインデントを解析
- **表検出**: 列の位置の揃いから表構造を推定
- **段落結合**: 行間と句読点から適切に段落を結合

### テンプレート

- **議事録**: 日時、場所、議題、決定事項などを自動強調
- **論文・レポート**: Abstract、Introduction、Methodなどの章立てを強調
- **請求書・帳票**: 発行元、宛先、明細、合計などを構造化

## プライバシーとセキュリティ

- ✅ すべての処理をブラウザ内で完結
- ✅ 外部サーバーへのデータ送信なし
- ✅ ネットワーク遮断状態でも動作（初回学習データ取得を除く）
- ✅ ローカルストレージへの保存は最小限
